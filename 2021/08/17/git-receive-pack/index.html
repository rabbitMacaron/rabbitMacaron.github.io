이클립스에서 git-receive-pack not permitted on 에러 발생 시 해결방법

Git에 push하다 갑자기 발생한 에러:bomb:

저번주까지 문제 없다가 갑자기 에러가 발생해서 너무 당황스러웠다..


에러메세지를 보고 이클립스에 저장해뒀던 Git ID/PW 정보를 다시 입력해보았지만 여전히 에러가 발생했다 ..



:heavy_check_mark:에러 원인

Git Hub에서는
Git 작업 인증 시, 암호를 사용하는 인증은 더 이상 사용할 수 없고
곧 작동하지않을거라는 메일을 보냈었는데 확인을 못 했다…:disappointed_relieved:


이 문제의 해결방법은
이클립스에 저장해뒀던 정보를 수정하는 것이 맞다.
다만, 비밀번호가 아닌 개인토큰을 입력해야한다





:key:해결방법

1.

Git에 로그인 한 후, 프로필 클릭 > Settings 에 접근한다.

2.

좌측에 있는 Developer settings 에 접근한다.

3.

(해당 사진 이후의 사진들은,
이미 토큰 생성 후 캡쳐했기 때문에 화면에 차이가 있다!!!)

좌측의 Personal access tokens > Generate new token 버튼을 클릭한다.

4.

Note : 토큰의 이름을 작성한다. (본인이 구분할만한 설명 혹은 용도를 작성하면 될 거 같다)
Select scopes : 항목들 중 부여할 권한을 선택한다

하단의 Generate token 버튼을 클릭한다.

5.

Personal Access Token이 생성되었다!!!
해당 화면과 다르게, token의 값이 화면에 보일 것이다.
생성된 토큰값은 다시 확인할 수 없으니 값을 복사해서 잘 보관해야한다

6.

이클립스로 돌아와서,
상단의 window > preferences 에 접근한 후, General > Security > Contents 를 클릭한다.
GIT을 펼쳐서 저장해뒀던 GIT ID/PW를 삭제(delete)한다.

7.

다시 Git에 Push 혹은 Pull을 시도할 경우,
Git ID/PW를 입력하는 창에서 ID와 personal token을 입력한다.



해결 완료:blush:
